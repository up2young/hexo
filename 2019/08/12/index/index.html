外部md文件测试

项目背景

因为学习与工作的需要，想要打造一个云笔记本，作为一名重度笔记用户，这非常重要，有时你需要在PC上长篇大论，有时你需要立刻用手机记录几句想法的和灵感

项目目标

  • 支持多终端，包括PC和移动端，方便随时随地进行记录
  • 对记录的格式化尽可能有最大自由度的支持，支持通用格式更佳，如markdown
  • 所有内容的数据完全个人可控
  • 支持对数据科学的学习需求：
    • 对代码编写的友好性，例如代码提示格式化展示等，最好还支持运行
    • 数据层与表现层分离
    • 支持版本管理
  • 分享与协作
    • 支持外网匿名用户查看
    • 支持多用户注册、使用与互动

初步思路

  • 可扩展多用户版：云服务器 + jupyterhub
  • 低成本单用户版：树莓派 + anaconda(或miniconda + jupyter)

小结

倒腾了一天，树莓派2B+成功运行了可远程访问的Jupyter，这么看来3代和4代即使本机运行也应该完全没有问题，如果是低版本的树莓派，建议尽量用有线网络以及高速卡减少损耗，4代支持USB3后非常值得期待，而jupyter一旦安装成功，相当于同时拥有了文件管理、笔记本、编程环境、数据探索环境以及terminal终端，且从性能来看，如果远程访问会大幅降低树莓派本机运行GUI带来的性能压力，即使是树莓派2代这样的硬件，轻量级使用也只是20-50%的CPU，简直就是树莓派必备良器，明天进一步看性能

但是如何使用大于3.4的Python版本呢？

在Raspberry Pi上向Miniconda添加Python 3.5 / 3.6 / 3.7

在安装Miniconda之后，我还没有安装比Python 3.4更高的Python版本，但我需要Python 3.5。这是我在Raspberry Pi 4上运行的解决方案：

首先，我添加了jjhelmus的Berryconda软件包管理器（这是一种最新版本的miniconda armv7l版本）：

conda config –add channels rpi
直到现在我才能安装Python 3.5,3.6,3.7，而无需自己编译：

conda install python=3.5
conda install python=3.6
conda install python=3.7
之后我能够使用添加的Python版本创建环境，例如使用Python 3.5：

conda create –name py35 python=3.5
现在可以激活新环境“py35”：

source activate py35
